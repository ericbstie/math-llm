### ATTACK 4 (the complexity / degree claim, most load-bearing): is {n, n+2} genuinely degree-1, and is "complexity 0" right?

**What must be true for L-C3.** The unification "obstructions = linear phases /
Dirichlet characters (degree 1)" requires that the two-point correlation `Σ_n f(n)
g(n+2)` be **controlled by the U² (degree-1) Gowers norm** and by nothing higher — so
that the inverse theorem's alternative is genuinely "degree-1 character" (= Siegel =
C6), not some higher-degree nilsequence. If the pattern were higher-degree, the
"C5↔C6 = Heath-Brown" identification and the degree-1 framing would be wrong.

**Test against the "true complexity" literature (Gowers–Wolf; Cauchy–Schwarz
complexity).** The literature (Gowers–Wolf "The true complexity of a system of linear
equations", arXiv:0711.0185; and the refinement line arXiv:2109.05965 / arXiv:2109.05731)
distinguishes:
- **Cauchy–Schwarz (CS) complexity** of a system of forms = smallest `s` such that each
  form can be separated from the others into `s+1` classes, so the average is controlled
  by `U^{s+1}`.
- **True complexity** = the smallest `s` such that `‖f‖_{U^{s+1}}=o(1)` forces the
  average `→ 0`; can be **strictly smaller** than CS-complexity.

For the **two-point** configuration — one free variable `n`, the two forms `ψ₁(n)=n`,
`ψ₂(n)=n+2` (the pair `{n, n+2}`) — the average is `(1/N)Σ_n f(ψ₁(n)) g(ψ₂(n))`.
A single Cauchy–Schwarz in the `n`-variable bounds this by `‖f‖_{U²}` (equivalently
`‖g‖_{U²}`): the two forms are separated into the `s+1=2` classes `{ψ₁}`, `{ψ₂}`, so
the **CS-complexity is 1**, and the average is controlled by the **U² (degree-1)
norm**. By the U² inverse theorem, the only degree-≤1 obstruction is a **linear phase /
character** — exactly the Siegel/C6 object. (The search-surfaced examples confirm the
framework's bottom end: the *nonlinear Roth* configuration has true complexity 0; a
genuinely linear two-point pattern sits at CS-complexity 1 / true complexity ≤ 1,
controlled by U².)

**Correction to EXPLORE's terminology (recorded, not fatal).** The EXPLORE prose
(Part 2.3) called the two-point pattern **"complexity 0"**. The precise invariant is
**CS-complexity 1, true complexity ≤ 1** — controlled by U², **not** U¹ (the U¹
"norm" is degenerate / not a norm; nothing nontrivial is controlled by it). "Complexity
0" in EXPLORE's sense meant "the bottom, degenerate end where the U-norm method names
but cannot rule out the obstruction"; the *correct* statement of that intuition is:
**the two-point linear correlation is the lowest non-trivial (U²/degree-1) case, and
this is exactly the parity-barrier face — the inverse theorem hands back a degree-1
character (= Siegel = C6) which it cannot itself rule out for a two-point pattern
because there is no third form to average over.** The conclusion EXPLORE drew is
**correct**; only the numeral "0" should read "1 (= U²/degree-1, the parity-barrier
face)". This is a terminology fix, not a flaw in the unification.

**Why the degree-1 control does NOT give a PROVIDE (the decisive point).** Being
controlled by U² is **not** the same as the U² being small. For `k ≥ 3`-term patterns
(complexity `≥ 1` with extra forms), the linear-forms machinery converts "controlled by
`U^{s+1}`" into an actual count via the *generalized von Neumann theorem plus an
average over the extra free variables*. For the **two-point** pattern there is **no
extra free variable to average over** — `{n, n+2}` has one variable — so "controlled by
U²" cannot be upgraded to "the U² is `o(1)`" without independently proving the U² is
small, which for `λ` is exactly the Chowla/parity statement. This is the higher-order
restatement of the parity barrier and it is **concordant with SO-L²**: the missing
ingredient is the average that the two-point pattern does not provide.

**FINDING — degree/complexity claim is CORRECT (with a terminology fix).** Two-point
{n, n+2} is genuinely **U²/degree-1** (CS-complexity 1, true complexity ≤ 1);
obstructions = linear phases / Dirichlet characters; the C5↔C6 = Heath-Brown =
degree-1-GTZ identification stands. "Complexity 0" → "complexity 1 (U²/degree-1, the
parity-barrier face)". No PROVIDE: U²-control ≠ U²-smallness for a two-point pattern.

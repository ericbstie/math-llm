## Adversarial Pass A — clause 1's stated involution is not Type-I invariant (computational counterexample)

**Target.** L1 clause 1 asserts: every Type-I nonnegative-weight scheme's leading-order
output is **invariant** under `a_n -> ½(1±λ(n))a_n`, `λ(n)=(-1)^Ω(n)`, and concludes no
such scheme certifies `π₂(x)→∞`. The constructive core (EXPLORE §2(d), Audit 2) is that
`(1+λ)a_n` and `(1-λ)a_n` are two nonnegative sequences with "the same Type-I data up to
level D but opposite parity weighting."

**Attack.** Take this literally on the actual twin object: apply the involution to the
twin-admissible sequence and measure whether the two twisted sequences in fact share the
Type-I (residue-class, level-D) data they are claimed to share. If the residue-class sums
differ, the stated involution is not the Type-I-invariant object and clause 1's stated
mechanism is wrong.

```python
import math
N = 2_000_000
Om = [0]*(N+1); is_comp=[False]*(N+1)
for p in range(2,N+1):
    if not is_comp[p]:
        pk=p
        while pk<=N:
            for m in range(pk,N+1,pk): Om[m]+=1
            pk*=p
        for m in range(2*p,N+1,p): is_comp[m]=True
is_prime=[False,False]+[not is_comp[n] for n in range(2,N+1)]
def lam(n): return 1 if Om[n]%2==0 else -1
x=N-2
z=50; small=[p for p in range(2,z+1) if not is_comp[p]]
def admissible(n):
    if n<1 or n+2>N: return False
    for p in small:
        if n%p==0 or (n+2)%p==0:
            if not (n==p or n+2==p): return False
    return True
def bplus(n):  return (1+lam(n))/2 if admissible(n) else 0
def bminus(n): return (1-lam(n))/2 if admissible(n) else 0
adm=[n for n in range(1,x+1) if admissible(n)]
Mp=sum(bplus(n) for n in adm); Mm=sum(bminus(n) for n in adm)
print(f"total mass: b_plus={Mp:.1f} b_minus={Mm:.1f} ratio={Mp/Mm:.5f}")
for d in [3,5,7,11,13]:
    for a in (1,2):
        idxs=range(a,x+1,d)
        sp=sum(bplus(n) for n in idxs); sm=sum(bminus(n) for n in idxs)
        if sm>0: print(f"  d={d:3d} a={a}: plus={sp:9.1f} minus={sm:9.1f} ratio={sp/sm:.4f}")
true_twins=[n for n in range(2,x+1) if is_prime[n] and n+2<=N and is_prime[n+2]]
print("true twins captured: by b_plus=%d by b_minus=%d"
      % (sum(bplus(n) for n in true_twins), sum(bminus(n) for n in true_twins)))
```

## Adversarial Pass B — the correct carrier's Type-I invisibility is OPEN (two-point Chowla), not a theorem

repaired carrier is in fact Type-I-invisible, and whether that invisibility is a theorem.

parity pair; its constructive half rests on an OPEN input. The honest content of clause 1
splits: (1a) the *dimension-1 model* statement (Bombieri) is a theorem; (1b) the transfer
to the *dimension-2 twin form* via the explicit parity pair needs the open two-point
Chowla bound. **The lemma's "provable, scoped" label over-promises on (1b).**

## Adversarial Pass C — "a fortiori dimension 2" and Type-II *necessity* for the binary form (EXPLORE attacks #2, #3)

**Target.** Clause 2 asserts detecting the twin pair **requires** a Type-II estimate,
citing the Friedlander–Iwaniec (FI) asymptotic-sieve dichotomy, with an "a fortiori,
dimension 2" step (the twin sequence has sieve dimension κ=2).

**Attack.** Pin whether (i) the FI dichotomy is a theorem and what its exact hypotheses
are, and (ii) whether "necessity of Type-II for `n(n+2)`" is *proven* or is "no Type-I
route is known."

**What it found.**
- The FI / Bombieri asymptotic sieve **is** a theorem (Bombieri 1976; Friedlander–Iwaniec,
  *Ann. of Math.* 148 (1998)): on an EH-type Type-I level-of-distribution hypothesis, for
  a sequence of **sieve dimension exactly 1**, Type-I data determines the generalized von
  Mangoldt asymptotics `Λ_k` for every `k≥2` (almost-primes) but **provably not** `Λ_1`
  (primes) without an additional **Type-II bilinear** estimate. The mechanism is the
  "equal weight" property: `Λ_k(n)=Σ_{d|n}μ(d)log^k(n/d)` gives asymptotically equal mass
  to `Ω`-even and `Ω`-odd numbers for `k≥2` (confirmed below, Pass E). This is well-cited.
- **The necessity is proven *for the dimension-1 axiomatized model*, via an explicit
  construction** (a sequence satisfying (B0)–(B2) with the prime count tilted by a
  multiplicative `±1` carrier). That is a genuine theorem.
- **But "a fortiori, dimension 2 is at least as obstructed" is NOT itself a theorem of FI.**
  FI's dichotomy is stated for κ=1; the twin sequence is κ=2. The intuitive "harder ⇒ at
  least as obstructed" is plausible but I found **no citation that proves the necessity of
  a Type-II estimate specifically for the dimension-2 binary form `n(n+2)`** as a formal
  impossibility theorem. What *is* documented is the weaker, honest statement: **no Type-I
  scheme is known to detect the twin pair, and the general "parity problem" is an open
  question as to whether fundamentally better (still Type-I-only) methods exist.** The
  Wikipedia/standard formulation is explicit: "The parity problem remains an **open
  question** in sieve theory regarding whether fundamentally better methods can be
  developed to overcome this inherent limitation." (Searches consistent across sources.)

**Consequence.** Clause 2's word **"requires" overstates** to a theorem what the
literature gives as: *Type-II suffices (FI) and Type-I is not known to suffice; necessity
is proven only in the dimension-1 model.* For `n(n+2)` (κ=2) the necessity is **not a
cited theorem** — it is the open parity question. The "a fortiori" step is a heuristic
bridge, not a proof. So clause 2's necessity claim, for the binary form, is **OPEN**, and
the honest reduction is "no Type-I route known + Type-II is the known escape route," not
"a Type-II estimate is provably necessary."

## Adversarial Pass D — exact reading of the Polymath8b GEH barrier (EXPLORE attack #4); GRH/EH/GEH scope check

**Target.** Clause 3: "no Type-I hypothesis, including GEH, supplies the Type-II input;
Polymath8b is the witness — under GEH the sieve bottoms out at `H₁=6`, parity-obstructed
below 6, never 2." I test the *exact* status of this barrier and whether the argument
secretly assumes more than is proven.

**What it found.**
- Polymath8b (D.H.J. Polymath, *Res. Math. Sci.* 1 (2014),
  [arXiv:1407.4897](https://arxiv.org/abs/1407.4897); retrospective
  [arXiv:1409.8361](https://arxiv.org/abs/1409.8361)) **does** prove `H₁≤6` under GEH and
  `H₁≤246` unconditionally. Correct as cited.
- The barrier "6 cannot be improved" is **method-specific and is not an absolute
  impossibility theorem.** The precise documented statement is: *the Selberg/parity
  obstruction prevents the **first-moment (Maynard–Tao) multidimensional sieve** from
  proving `H₁<6` under GEH.* Authoritative qualifier found verbatim in summaries: the
  parity obstruction "prevents any better bound on `H₁` from **purely sieve-theoretic**
  considerations," and improvements beyond 6 "**cannot be overcome by the first-moment
  Maynard sieve methods alone.**" So clause 3 is correct only if it says **"this maximal
  Type-I first-moment machine stops at 6,"** not "Type-I is absolutely insufficient."
- **Crucial nuance that L1 currently omits:** the barrier at 6 is tied to **GEH
  specifically.** Under the *weaker* plain EH, "the parity obstruction does **not exclude**
  the possibility that one could achieve better bounds" (Polymath8b / Tao). I.e. the clean
  "stops at 6" statement is a GEH statement; it is not even known to be the EH barrier.
  This makes clause 3 *more* method-and-hypothesis-specific than L1 presents.
- **GRH/EH/GEH scope.** L1 does **not** secretly assume GRH: its clauses are explicitly
  conditional-class statements and the unconditional facts cited (Zhang, Maynard,
  Polymath8b's 246) do not use GRH. Clause 3 *uses* GEH only as the strongest Type-I
  hypothesis to show "even GEH doesn't help below 6," which is the correct logical
  direction (it does not assume GEH to prove anything positive). **No hidden GRH/EH
  over-assumption found.** The one correction is the GEH-vs-EH scoping above.

**Consequence.** Clause 3 is **citation-backed but must be re-scoped**: it witnesses that
the *first-moment multidimensional sieve under GEH* stops at gap 6; it does **not** witness
"Type-I is provably insufficient in any absolute sense." As written ("no Type-I hypothesis
supplies the Type-II input") it is a fair informal gloss but is not a theorem — it is the
*expected* reading of a method-specific barrier. Re-scoped to the Polymath8b statement, it
is ESTABLISHED-IN-LITERATURE; as an absolute claim it is OPEN.

## Adversarial Pass E — counterexample hunt (EXPLORE attack #6) and circularity (attack #5)

**Target (#6).** Does **any** unconditional theorem give a positive lower bound for a
**fixed** prime pair (e.g. `π₂(x)→∞`) using only Type-I inputs? If yes, clause 1 is
FALSIFIED outright.

**What it found.** **No such theorem exists in the literature.** The strongest
unconditional Type-I results are: Chen (infinitely many `p` with `p+2 ∈ P₂`) — *not* a
fixed prime pair; Maynard/Zhang/Polymath8b (bounded gaps, `≥2` of `k` entries prime) —
never a *fixed* pair; the FI/Heath-Brown parity escapes (`a²+b⁴`, `x³+2y³`) — these
**do** count a fixed prime form but **only by injecting a Type-II bilinear estimate**, and
they have **no known analogue for `n(n+2)`** (no thin-set bilinear structure). So clause 1
is **not falsified by a known counterexample** — there is no unconditional Type-I lower
bound for a fixed prime pair. (This is consistent with TPC being open: if such a Type-I
theorem existed it would essentially solve TPC.) **Clause 1 survives the
counterexample hunt** — its failure mode is the over-statement found in Passes A/B, not a
counterexample.

**Target (#5, circularity).** Is clause 3 ("a Type-I method fails to reach 2") just a
restatement of clause 1 ("Type-I can't detect primes")? **Partly yes.** Clause 3 is an
*instance* of the clause-1 phenomenon (the Polymath8b barrier *is* the parity obstruction
manifesting in the multidimensional sieve), so it is **corroborating, not independent,
evidence.** The lemma is not viciously circular — clause 3 is a concrete optimized
variational witness, which has independent value — but it should be presented as "the same
obstruction, instantiated and made quantitative," not as a logically independent third
pillar. (Minor; recorded for EXPLORE.)

**Supporting computation (the genuine "equal weight" mechanism is asymptotic, slow, and
does not certify clause 1 at accessible ranges).**

```python
import math
N=200000
Om=[0]*(N+1); is_comp=[False]*(N+1)
for p in range(2,N+1):
    if not is_comp[p]:
        pk=p
        while pk<=N:
            for m in range(pk,N+1,pk): Om[m]+=1
            pk*=p
        for m in range(2*p,N+1,p): is_comp[m]=True
def factor(n):
    f={}; d=2; m=n
    while d*d<=m:
        while m%d==0: f[d]=f.get(d,0)+1; m//=d
        d+=1
    if m>1: f[m]=f.get(m,0)+1
    return f
def mob(n):
    if n==1: return 1
    f=factor(n)
    return 0 if any(e>1 for e in f.values()) else (-1)**len(f)
def divisors(n):
    ds=[]; d=1
    while d*d<=n:
        if n%d==0:
            ds.append(d)
            if d!=n//d: ds.append(n//d)
        d+=1
    return ds
def Lam2(n): return 0.0 if n==1 else sum(mob(d)*(math.log(n/d))**2 for d in divisors(n))
total=sum(Lam2(n) for n in range(2,N+1))
tilt =sum(Lam2(n)*(1 if Om[n]%2==0 else -1) for n in range(2,N+1))
print(f"sum Lambda_2 = {total:.1f}; lambda-tilt = {tilt:.1f}; ratio = {tilt/total:.5f}")
```

**The difficulty bottoms out at one sharp OPEN claim** — exactly state (3). After

## Adversarial Pass F — the uniformity test: is "bucket-ratio ≈ 0.0048" a fixed-`d` artifact that blows up as `d→D`?

**Target.** The Apply step (§2.3 of `work/1781354180-EXPLORE.md`) argued: every Type-I
datum `T_d` is parity-invisible (the carrier `λ(n)λ(n+2)` has worst bucket-ratio ≈ 0.0048
across `d ∈ {2,…,13}`), so every signed combination is invisible, so Chen's subtraction
cannot escape. **But the load-bearing word is *uniform*.** "Invisible enough to kill the
leading term" means `∑_{n≤x, n≡a (d)} λ(n)λ(n+2) = o(count)` *uniformly over `d ≤ D =
x^{1−ε}`*. The Apply computed the ratio only at small fixed `d`, where the count per bucket
is enormous (`~x/d ≈ 10⁶`). The whole question is what happens as `d` grows toward `D`,
where the count per bucket shrinks toward `x^ε`.

**Attack.** Sweep `d` from 2 up to 10⁵ at `N = 8·10⁶`. For each `d`, over residue classes
`a mod d`, record (1) the worst `|bucket sum|/count` — the quantity that must be `o(1)` for
the "invisible enough to kill the leading term, uniform in `d`" claim — and (2) the worst
`|bucket sum|/√count` ("z-score"), which stays bounded under mere square-root cancellation.
If (1) blows up as `d` grows, the "0.0048 invisibility" is a small-`d` artifact and the
uniform statement is **not** witnessed by the numerics — it is the open conjecture.

```python
import math

# Sieve Omega(n) (number of prime factors with multiplicity) up to N
N = 8_000_000
Om = [0]*(N+1)
is_comp = [False]*(N+1)
for p in range(2, N+1):
    if not is_comp[p]:
        pk = p
        while pk <= N:
            for m in range(pk, N+1, pk): Om[m]+=1
            pk *= p
        for m in range(2*p, N+1, p): is_comp[m]=True

def lam(n): return 1 if Om[n]%2==0 else -1
x = N-2
def carrier(n): return lam(n)*lam(n+2)

S = sum(carrier(n) for n in range(1,x+1))
print(f"GLOBAL  sum lambda(n)lambda(n+2), n<=x={x}:  {S}   ratio/x = {S/x:+.6f}")
print(f"        (for comparison, 1/sqrt(x) = {1/math.sqrt(x):.6f})")
print()
print("d-sweep: worst bucket over residue classes a mod d")
print(f"{'d':>6} {'count/bkt':>10} {'worst|sum|/cnt':>15} {'worst|sum|/sqrt(cnt)':>22}")
test_ds = [2,3,5,7,11,13,30,210,1000,2310,10000,30030,100000]
for d in test_ds:
    sums=[0]*d; cnts=[0]*d
    for n in range(1, x+1):
        r=n%d; sums[r]+=carrier(n); cnts[r]+=1
    wr = max(abs(sums[a])/cnts[a] for a in range(d) if cnts[a])
    wz = max(abs(sums[a])/math.sqrt(cnts[a]) for a in range(d) if cnts[a])
    print(f"{d:>6} {x/d:>10.1f} {wr:>15.5f} {wz:>22.3f}")
```

**What it found.** The "0.0048" invisibility is **a small-`d` artifact.** The worst

## Adversarial Pass G — confirm the per-bucket cancellation is genuine (√-cancellation, not anomalous bias), isolating that ONLY uniformity is open

import math

## Adversarial Pass H — separate the three statements (i)/(ii)/(iii); is (i)→(iii) or (ii)→(iii) a theorem?

**Target.** The Apply (and C1's audit) leans on "the parity obstruction (C2) applies
verbatim." I check whether the chain from the established facts to **(iii)** "no signed
linear functional of Type-I data for `n(n+2)` yields `π₂→∞`" is rigorous, or whether each
link is either a heuristic or a dimension-1-only theorem.

**Attack (literature, verified this pass; no invented citations).**

**(i) The principle "sieves can't break parity."** Authoritative wording (search-returned
summary of Tao, *Open question: The parity problem in sieve theory*, 2007, and the
*Parity problem* references): *"This obstruction is ultimately powered by the **Möbius
pseudorandomness principle** that asserts that the Möbius function is asymptotically
orthogonal to all 'structured' functions. **This is a heuristic principle rather than a
proven theorem** — it explains why sieve methods face fundamental limitations."* Selberg's
original statement is a limitation **demonstrated by a model construction** (a set all of
whose elements have an odd number of prime factors, on which sieve upper/lower bounds are
off by a factor of 2), **not** a universal impossibility theorem. Tao's 2014 post *"A
general parity problem obstruction"* likewise *constructs model weight systems / sequences*
with prescribed Type-I behaviour — again a limitation-by-construction, not a theorem that
no functional whatever can succeed for a *specified* sequence like `n(n+2)`.
**Verdict: (i) is a heuristic principle. (i) → (iii) is NOT rigorous.**

**(ii) Bombieri's asymptotic-sieve parity theorem.** This *is* a real theorem (Bombieri
1976; Friedlander–Iwaniec, *Asymptotic sieve for primes*, Ann. of Math. 148 (1998)). But
its hypothesis is **sieve dimension exactly 1**: search-returned summary of Tao's *Notes on
the Bombieri asymptotic sieve* (2016) states *"the product for H converges to a nonzero
constant **if and only if `g` has sieve dimension 1**,"* and *"the obstruction to the parity
problem is only one-dimensional (on EH)."* The twin form `F(n)=n(n+2)` has **sieve dimension
`κ = 2`** (two residue classes removed per prime). FI break parity in their setting by
**adding a bilinear-form (Type-II) estimate as an extra axiom** (their argument starts from
Vaughan's identity with a bilinear error term that must be controlled), and apply it to a
**thin** sequence (`a²+b⁴`) that *has* the bilinear variable; they do **not** prove a
dimension-2 impossibility theorem, and `n(n+2)` has no such bilinear structure (this is C3).
**Verdict: (ii) is a theorem, but only in dimension 1. (ii) → (iii) is NOT rigorous** (it is
the "a fortiori dimension 2" heuristic already quarantined in the GRAVEYARD as
`[dim-2-necessity-as-theorem]`).

**(iii) The specific claim for `n(n+2)`.** No cited theorem proves it. Its *quantitative*
form — "every signed linear functional `∑_d c_d T_d` of the Type-I data is perturbed by
`o(main term)` under the carrier `λ(n)λ(n+2)`, uniform over `d ≤ D`" — reduces (Pass F/G) to
`∑_{n≤x, n≡a(d)} λ(n)λ(n+2) = o(x/d)` uniform in `d ≤ D`, i.e. the **unaveraged shift-2
two-point Chowla estimate in arithmetic progressions**. Only the *logarithmically averaged*
two-point Chowla is a theorem (Tao 2015, arXiv:1509.05422, Forum Math. Pi 4 (2016); Pilatte
arXiv:2310.19357 improves only the log-averaged bound). **Verdict: (iii) is OPEN. It is C5.**

**Consequence.** The "necessity" half of the project's framing — "C1 is comprehensive *and
parity-blocked*, so the class provably cannot do it" — rests on **(iii)**, which is neither
**(i)** (heuristic) nor **(ii)** (dimension 1). It is the **same open object as C5**. So:

- C1's *definitional* content (which methods are signed linear functionals of Type-I data:
  Brun, Selberg, GPY, Maynard–Tao, Chen via subtraction) **survives as a NEEDS-REVIEW
  definition** — that part is genuinely just bookkeeping a human can check.
- C1's *"hence parity-blocked / comprehensiveness proves the wall"* content **is not a
  separate established fact**; it is C5. It must be demoted to **OPEN** and **merged into
  C5**, not cited as a closed audit.

The previous EXPLORE step's claim that "C1's comprehensiveness is now audited" is **correct
only for the membership half** (Chen is a signed functional — that linear-algebra is sound).
The *parity-blocking* half it appeared to also close is exactly the open crux. This pass
removes the hidden circular dependency by re-layering.
